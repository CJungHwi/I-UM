# 프로젝트 개발 상태 분류

본 문서는 업로드된 기획 명세서(새 텍스트 문서.txt)의 **순서를 그대로 따라** 현재 리포지토리(`/workspace`)의 구현 상태를 "✅ 개발 완료 / ⚠️ 부분 구현(플레이스홀더 UI만) / ❌ 미개발"로 분류한 기록입니다.

> 판단 근거
> - **페이지**: `src/app/<경로>/page.tsx`
> - **서버 액션**: `src/actions/*.ts`
> - **DB 프로시저**: `prisma/procedures/*.sql`
> - **사이드바 메뉴**: `prisma/procedures/menu.sql` 의 시드(`auto_menu`)
>
> "⚠️ 부분 구현"은 라우트/페이지 스캐폴드(Card 레이아웃과 설명 문구)만 존재하고, 서버 액션·DB 연동·도메인 로직은 아직 없는 상태를 의미합니다.

## 파트 C — 각 카테고리별 워크플로우 / 프로세스 정의 (첨부파일 "1. CRM ~ 7. BI" 프로세스 섹션)

> 기획서의 각 프로세스(`[P1-1] ~ [P7-2]`) 순서를 그대로 유지합니다.
> 입력(Input)·처리(Process)·출력(Output)은 기획서 기준이며, "상태"는 해당 프로세스가 현재 리포지토리에서 실제로 흐르는지를 평가한 것입니다.

### 1. CRM — 고객 확보 및 전환 워크플로우
핵심 목표: 신규 문의를 원생 고객으로 전환시키는 '퍼널(Funnel)' 관리.

| 순번 | 프로세스 | 상태 | 현황 |
|:---:|:---|:---:|:---|
| P1-1 | 상담 접수 등록 및 상태 관리 (접수 일시 → 상담 카드 → 상담 상태값 관리) | ❌ | `crm/prospects` 페이지는 플레이스홀더. 상담 DB/상태 전이 없음. |
| P1-2 | 레벨 테스트 → 반 매칭 (자동 커리큘럼 매칭, 반 정원 확인, 반 배정) | ❌ | 레벨 테스트 도메인·반 배정 로직 없음. |
| P1-3 | 이탈 조짐 감지/상담 재개 (예: 3주 이상 결석 시 상담 알림) | ❌ | 출결 도메인이 없어 감지 룰 구동 불가. |

### 2. LMS — 학습 데이터 기반 관리 워크플로우
핵심 목표: 학생의 학습 데이터를 수집하고 게이미피케이션으로 동기를 부여.

| 순번 | 프로세스 | 상태 | 현황 |
|:---:|:---|:---:|:---|
| P2-1 | 등·하원 기록 & 안심 알림 (실시간 등원 기록 → 셔틀·푸시 연계) | ❌ | 출결 입력/알림 트리거 없음. (알림 발송 인프라는 준비됨 — `notifications.sql`) |
| P2-2 | 과제/성적/반 결과 공유 (단원별 성적 그래프, 과제 진행 트래킹) | ❌ | 데이터 입력/집계 없음. |
| P2-3 | 게이미피케이션 포인트 지급 (출결·과제·성적 목표 달성 시 자동 포인트 지급) | ✅ | `gamification-actions.ts` 에 `triggerAttendancePoint`, `triggerAssignmentAPoint`, `grantManualPoint` 구현. `lms/gamification` 화면에서 호출. **단, 트리거 소스(출결/과제 입력)가 없어 현재는 UI 버튼으로만 흐름이 돌아감.** |

### 구현 완료된 핵심 자산
- **인증 · RBAC**: NextAuth + `ADMIN/USER`·`userLevel`·`academyId` 게이팅 (`src/auth.ts`, `src/lib/ium-user.ts`, `rbac.sql`).
- **회원가입 / 소속 학원 관리 / 사용자 승인**: `src/app/admin/{register,academies,users}/` + `ium-user-actions.ts`, `ium-academy-actions.ts`, `ium_academies_admin.sql`, `ium_users.sql`.
- **알림 시스템(6-1 / P6-1 기반)**: 템플릿·범위·수신자 지정 발송, 미읽음 카운트 (`notifications.sql`, `notification-actions.ts`, `notifications-admin-client.tsx`).
- **스레드 타임라인(6-2 / P3-1)**: 학생별 담당자 스레드 (`student_threads.sql`, `thread-actions.ts`, `crm/threads/*`).
- **게이미피케이션(+1 / P2-3)**: 포인트·배지·히스토리·수동 지급·Confetti (`gamification.sql`, `gamification-actions.ts`, `lms/gamification/*`).
- **메뉴/사이드바 시스템**: `get_menu(userLevel)` 프로시저 + `auto_menu` 시드로 **기획서의 7개 카테고리 메뉴 트리가 이미 노출** (`menu.sql`).

### 미개발 우선순위가 높은 블록
1. 상담(P1-1) · 레벨 테스트/반 매칭(P1-2) · 학생/가족 마스터(1-5) — **다른 모든 워크플로우의 기준 데이터**.
2. 출결 입력(2-1 / P2-1) — 게이미피케이션 트리거, 셔틀 알림, 퇴원 예측의 공통 소스.
3. 수강료/청구(4-1 ~ 4-3 / P4-1 ~ P4-3) — 구독 결제·형제 할인·미납 독촉이 한 세트.
4. 성적/과제 입력(2-3 / 2-4) — 학습 슬럼프 감지(+5) 및 자동 알림장(P6-1)의 입력 소스.
5. 셔틀 GPS(5-3 / +6 / P5-1) — 학부모 신뢰 확보의 핵심 UX.
