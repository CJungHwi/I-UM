# 프로젝트 개발 상태 분류

본 문서는 업로드된 기획 명세서(새 텍스트 문서.txt)의 **순서를 그대로 따라** 현재 리포지토리(`/workspace`)의 구현 상태를 "✅ 개발 완료 / ⚠️ 부분 구현(플레이스홀더 UI만) / ❌ 미개발"로 분류한 기록입니다.

> 판단 근거
> - **페이지**: `src/app/<경로>/page.tsx`
> - **서버 액션**: `src/actions/*.ts`
> - **DB 프로시저**: `prisma/procedures/*.sql`
> - **사이드바 메뉴**: `prisma/procedures/menu.sql` 의 시드(`auto_menu`)
>
> "⚠️ 부분 구현"은 라우트/페이지 스캐폴드(Card 레이아웃과 설명 문구)만 존재하고, 서버 액션·DB 연동·도메인 로직은 아직 없는 상태를 의미합니다.

 - 학부모 전용 Web Push 구독 (VAPID 키) 또는 카카오 알림톡(템플릿 사전 심사)
 - `sp_shuttle_boarding_mark` 호출 시 후속으로 `sendNotificationRaw` 자동 호출 → 학부모(`targetUserId`) 지정 발송.

### +7. 상담 내용 키워드 분석 (❌)
- **DB**:
 - 상담(1-1) 본문을 N-gram 추출: 간단형은 프로시저 `sp_consult_tokenize_ngrams` + `ium_consult_keyword` 집계 테이블
 - 고도화: LLM Embedding(OpenAI/Cohere)으로 토픽 클러스터링
- **UI**: 워드클라우드 + 이탈 사유 Top-N 카드.
- **Ext**: LLM 기반 선택 시 API 키 필요.
